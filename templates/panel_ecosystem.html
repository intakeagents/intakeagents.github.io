Integration Architecture
How agents connect to your systems
Agents read from and write to your existing intake, scheduling, and billing platforms — no rip-and-replace required.
📋 Intake / Case Management
Brightree, WellSky, Kareo, or your internal system
ReadPull existing referral data — no re-keying
WritePush completed episode record when routed
UpdateLog gaps, outreach events, status changes in real time
📅 Scheduling System
Dispatch, vendor scheduling, or calendar platform
ReadCheck available windows before routing
WriteCreate appointment hold when episode is route-ready
UpdateConfirm or release hold after case manager response
🔌 Integration Method
Depends on your platform's capabilities
APIPreferred — direct integration via REST/FHIR endpoints
RPAFallback for legacy systems with no API layer
FileFlat file / HL7 export for closed platforms
Key question for your team: Does your intake system expose an API? Most modern platforms (Brightree, WellSky) do — that's a direct 2-week integration. Legacy or closed systems use RPA as a bridge. Either way, the agent architecture doesn't change — only the output destination.
Human in the Loop
Agents handle the straight-through path. Humans decide at every flagged exception.
Routine processing — extraction, validation, outreach, posting, reconciliation — runs autonomously. Human review is triggered at every flagged exception: ICD conflicts above a confidence threshold, denials requiring clinical judgment, appeals needing legal review, payer disputes that require negotiation. The agent escalates with full context and a recommended action. The human decides. That boundary is explicit, logged, and configurable per your clinical and compliance requirements. You set the thresholds. The agent respects them.
