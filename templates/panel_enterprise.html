Enterprise Readiness

Every question a technology leader asks before saying yes — integration, security, compliance, infrastructure, timeline, and support.

Key Considerations
🔗
Integration
How does it connect to our fax system / intake portal?
API-based or file-drop integration — we connect to your existing fax gateway (eFax, Sfax, RightFax) via webhook or SFTP folder watch. No rip-and-replace.
Does it plug into Salesforce, Guidewire, or our case management system?
Yes — REST API integration with standard WC platforms. Episode records push via webhook or batch export. Custom connectors built during integration discovery (Weeks 1–2).
API or file-based?
Both supported. API-first recommended for real-time routing; file-based available for legacy systems with no API exposure.
🔒
Security & Compliance
HIPAA/PHI — where does patient data live?
Patient data lives in your environment. Episode records written to your storage. PHI never persisted outside your boundary.
Is PHI passing through the LLM provider's API?
Yes — PDF text is sent to the LLM for extraction. LLM provider BAA is available and required before deployment. Provider does not train on API data under BAA.
Data retention and audit log requirements?
Configurable retention policies. Every decision logged with timestamp, model version, confidence, and reasoning — immutable audit trail per episode.
🖥️
Infrastructure
Cloud or on-premise?
Both. Containerized Python (Docker) runs on AWS, Azure, GCP, or on-premise. No cloud lock-in. Deployed to your existing Kubernetes or VM infrastructure.
What does it run on?
Python + Flask, containerized via Docker. Requires 2 vCPU / 4GB RAM minimum for 20-worker batch processing. Scales horizontally.
Can it handle our volume?
75 referrals in 60 seconds on a single instance. Horizontal scaling handles 10,000+ referrals/hour. LLM API rate limits are the ceiling — higher tiers handle 1,000+ RPM.
🚨
Support & Monitoring
Who maintains it after go-live?
Your team owns the deployed instance with full source code and runbooks. We provide 90-day hypercare support post go-live. Ongoing SLA available.
What happens when the LLM API goes down?
Automatic fallback to manual queue. Referrals queued, staff notified, processing resumes when API recovers. Zero data loss — episode state persisted before each API call.
Observability and alerting?
Built-in observability dashboard (Panel 4). Integrates with your existing monitoring stack — Datadog, Grafana, CloudWatch. Alerts on error rate, latency, and escalation spikes.
