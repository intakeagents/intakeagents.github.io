Architecture

How the Agent Works

Follow the agent step by step — what it sees, what it decides, and what it does at every stage from referral arrival to route-ready episode.

📠 Fax arrives
🎤 Voice call comes in
💻 Portal upload received
1
Channel Intake
Agent receives the referral
"A fax just arrived. Three documents — referral form, clinical notes, prescription. Agent reads every page before forming any opinion. If it's a voice call, Agent answers the line directly and asks the right questions in real time."
📄Fax: Agent ingests all 3 PDFs — referral form, clinical notes, prescription — and counts every page
🎤Voice: Agent answers the live intake line, conducts a structured conversation, and captures 14 fields in real time — 107 seconds average
📋Both channels feed the same queue automatically — same process, same output
2
AI Extraction — Large Language Model
Agent extracts 25 structured fields
"Agent reads all three documents simultaneously and extracts every field — patient name, ICD codes, HCPCS, physician NPI, adjuster contact, delivery address. Then Agent cross-references the documents against each other. If the referral form says M23.611 but the clinical notes say S83.209A — Agent catches it and flags the conflict with its reasoning."
🧠25 fields extracted: patient, claim, ICD-10, HCPCS, physician NPI, adjuster, delivery address, auth ref, language, transportation
⚠️Agent compares ICD codes across all documents — mismatches flagged with document-level reasoning, not just a generic error
📊Agent scores its own confidence — 0 to 100%. If confidence drops below 80%, Agent does not guess
3
Intelligence Layer — OWL Knowledge Graph
Agent validates every code against CMS rules
"Agent runs every ICD code and HCPCS through 24 deterministic rules sourced directly from CMS guidelines. These rules don't estimate — they confirm or reject with 100% confidence. If G89.11 appears on a post-surgical case, Agent rejects it and tells you exactly why: ICD-10-CM §I.C.6.b says post-surgical pain is G89.18. No ambiguity."
🧩24 CMS rules checked: ICD-10-CM FY2026 Guidelines, LCD L33803, HCPCS Level II — every rule cites its source
🛡️Four-layer defense: document cross-reference → KG validation → confidence threshold → human gate — Agent escalates before it guesses
⚖️Agent detects post-surgical context, bariatric weight flags, laterality conflicts, and authorization requirements automatically
4
Completeness Check
Agent classifies every gap by criticality
"Agent doesn't treat all missing fields the same. A missing auth reference number is a hard block — the episode cannot move without it. A missing transportation preference is required but doesn't hold the episode. Agent separates these instantly and acts on each differently."
🔴Hard blocks: auth_ref, appt_window — Agent holds the episode until resolved, nothing dispatched
🟡Required gaps: transportation, language — Agent collects in parallel, no delay to routing
✅Clean episodes skip outreach entirely — Agent routes immediately with no human touch
📧 Agent sends email outreach
"Agent drafts a specific email to the adjuster by name — not a template. It lists exactly which fields are missing, references the claim number, and sets a 24-hour deadline. Sent and logged the moment it dispatches."
• Addressed to adjuster by name — Linda Torres, not "Dear Case Manager"
• Lists specific missing fields with context — not a generic checklist
• Logged on episode record the moment it sends
📞 Agent places outbound voice call
"No email response after 24 hours. Agent places the call itself — 84 seconds. Confirms auth reference, collects appointment window, catches the bariatric spec the prescription missed. All logged."
• Auto-triggered after 24hrs with no email response
• Agent asks only about remaining gaps — doesn't re-ask resolved fields
• Transcript logged, episode fields updated from call response
5
Human Review Gate
Agent routes to specialist when it can't resolve
"When confidence falls below 80% — Agent stops. It doesn't guess on clinical decisions. It routes the episode to a specialist with the full conflict detail, the source documents, and its own reasoning. One click to approve or override. That decision is logged with the reviewer's identity."
🚨ICD conflicts below 80% confidence → specialist queue, episode blocked from dispatch
📝Specialist sees Agent's full reasoning — not just a flag, but why and what evidence Agent used
✅Approve or override in one click — decision logged with reviewer identity and timestamp
6
Route Output
Agent locks the episode and dispatches
"Every field populated. Every gap resolved — by email, by call, or by Agent self-correcting from clinical evidence. Agent locks the episode record, saves the full audit trail, and triggers vendor dispatch. The DME item ships with the right equipment, the right code, and the right patient."
💾Episode record saved — 25 fields, KG validation results, ICD conflicts, email body, call transcript
📤Vendor dispatch triggered — DME item, HCPCS, delivery address, auth ref all confirmed
📊Full audit trail available — every decision, every confidence score, every rule that fired
