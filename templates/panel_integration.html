Integration Method — Adapts to Your Platform
API
REST / FHIR Direct
Direct integration via platform API endpoints. Bidirectional, real-time, lowest latency. Most modern DME and billing platforms expose REST or FHIR APIs.
Preferred — 2-week integration for platforms like Brightree, WellSky, Kareo
RPA
Robotic Process Automation
UI-level automation as a bridge for legacy or closed systems with no API layer. Replicates human navigation precisely — same outcome, fully automated.
Fallback for legacy systems — no API required, no system changes needed
File
Flat File / HL7
Scheduled file-based exchange for fully closed platforms. Supports CSV, fixed-width, HL7 v2, and ANSI X12 EDI formats. Common in older payer integrations.
For platforms that only support batch file exchange
Key question for your tech team: Does your intake system expose an API? Most modern platforms (Brightree, WellSky, Kareo) do — that's a direct 2-week integration. Legacy or closed systems use RPA as a bridge. Either way, the agent architecture doesn't change — only the output destination. You own the credentials, the data never leaves your environment.
What this means for deployment: We map your current platform stack in the first week. Integration method is decided per system. Agent chain deploys against your environment — not a shared SaaS. Your IT team controls access, audit trail, and data residency from day one.
